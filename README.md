# ESA 数据保存工具使用说明

## 1. 项目用途

本项目用于通过 `pyvisa` 控制 ESA 频谱仪（E4407B），从仪器中导出**当前屏幕上扫描迹线**测量数据并保存为本地 `CSV` 文件。(如需保存特定频率范围或增加减少扫频点数，请在ESA上设置并显示在当前屏幕上)

项目中已经提供可直接运行的程序：

- 可执行文件：`dist\esa_datasave.exe`
- VISA 后端下载说明：`VISA后端\下载说明.txt`
- 示例数据：`demo_data.csv`
- 源码参考：`esa_datasave.py`

## 2. 使用前准备

在运行程序前，请先完成以下准备工作：

1. 将 ESA 频谱仪与电脑正确连接。
2. 确认电脑已经具备可用的 VISA 后端环境。
3. 如果电脑尚未安装 VISA 后端，请先下载并运行VISA后端：

建议安装完成后重启电脑，再连接设备进行测试。

## 3. 程序运行方式

启动安装好的VISA后端软件（**Keysight Connection Expert**）并确认连接正确，visa地址正确：

![本地截图](.\images\image.png "VISA后端示例")

本项目给后续使用者的推荐方式是直接运行打包好的可执行文件：

`dist\esa_datasave.exe`

运行后会弹出一个简单界面，需要填写以下内容：

- 保存路径：本地电脑上用于保存 `csv` 文件的文件夹路径（建议直接在地址框复制）
  ![alt text](.\images\image-1.png)
- 文件名：导出的文件名，不需要手动输入 `.csv`

填写完成后，点击界面中的执行按钮，程序会自动：

1. 通过 VISA 连接 ESA；
2. 在仪器内部生成一个 `C:\文件名.csv`（如不需要留存在机器内部可自行删除）；
3. 从仪器读取该 `csv` 数据；
4. 将数据保存到你填写的本地路径中。

执行成功后，会弹出提示框提示保存完成。

## 4. 一个典型示例

如果填写内容如下：

- 保存路径：`D:\ESA_Data`
- 文件名：`test1`

那么程序会在本地生成：

`D:\ESA_Data\test1.csv`

项目中的 `demo_data.csv` 就是一个导出结果示例，可用于查看文件格式。

## 5. 当前程序的重要说明

后续使用者需要特别注意以下几点：

1. 当前源码中仪器 VISA 地址是写死的：

    `GPIB0::18::INSTR`

    也就是说，程序默认按这个地址去连接 ESA。

2. 界面中输入的“保存路径”是本地电脑保存文件的位置，不是仪器地址。

3. 如果仪器实际地址不是 `GPIB0::18::INSTR`，则需要修改源码 `esa_datasave.py` 中这一行后重新打包，或者由维护人员提供新的可执行文件。

4. 程序依赖仪器能够响应相关 SCPI/VISA 命令，并允许读取仪器内部生成的 `csv` 文件。

## 6. 常见问题排查

本程序打包的exe文件具有未隐藏的命令行界面，任何程序报错可在命令行界面读取分析

### 无法连接仪器

请检查：

- VISA 后端是否已正确安装；
- GPIB/USB/LAN 等物理连接是否正常；
- 仪器是否开机；
- 仪器地址是否确实为 `GPIB0::18::INSTR`。

### 程序运行后没有保存文件

请检查：

- 填写的保存路径是否真实存在；
- 当前用户是否有该文件夹写入权限；
- 文件名中是否包含 Windows 不允许的特殊字符。

### 生成的数据内容异常

请先确认：

- ESA 当前测量状态正常；
- 仪器内能够正常存储和导出 Trace 数据；
- 仪器型号及命令兼容当前脚本。

## 7. 给维护人员的补充说明

源码文件为 `esa_datasave.py`，核心流程如下：

1. 使用 `pyvisa.ResourceManager()` 创建设备管理器；
2. 连接 `GPIB0::18::INSTR`；
3. 发送命令 `:MMEM:STOR:TRAC TRACE1,'C:\文件名.csv'`；
4. 发送命令 `:MMEMory:DATA? 'C:\文件名.csv'` 读回数据；
5. 将返回内容拆分后写入本地 `csv` 文件。

如需调整设备地址、界面文字或保存逻辑，可直接修改 `esa_datasave.py`
